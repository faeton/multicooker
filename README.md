# multivarka

Запускаешь несколько LLM-агентов одновременно решать одну задачу в
своих изолированных папках. Они работают параллельно, читают одну
общую папку `raw/` со справочными материалами, выгружают результат, а
потом судья (тоже LLM) сравнивает и выставляет оценки.

> "Мультиварка": одна задача, много блюд готовятся параллельно в
> своих чашах.

## За 30 секунд

```bash
# 1. Установить
cd ~/Sites/multivarka && pip install -e .

# 2. Создать новую варку
multivarka new my-task
cd cooks/my-task

# 3. Описать задачу
$EDITOR BRIEF.md          # что должны сделать участники
cp ~/some-reference.pdf raw/   # справочники, которые им можно читать
$EDITOR brief.yaml        # выбрать участников (claude/codex/gemini), таймаут

# 4. Готовить
multivarka cook my-task   # все участники работают параллельно

# 5. Судить
multivarka judge my-task
multivarka report my-task   # печатает leaderboard.md
```

## Что внутри

- **Host-mode (по умолчанию)** — multivarka запускает локально установленные
  CLI участников (`claude`, `codex`, `gemini`) с `--add-dir` указывающим
  на их изолированную рабочую папку. Использует подписочную
  авторизацию (Keychain / OAuth-файлы), поэтому работает без API-ключей.
- **Docker-mode** (флаг `--docker`) — каждый участник в отдельном контейнере
  (Dockerfile-ы готовы), требует API-ключей.

Подробности устройства, правила, ловушки и lessons learned — в
[HOWTO.md](HOWTO.md).
