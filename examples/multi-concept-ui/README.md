# multi-concept-ui - three divergent UI directions

Use this shape when you do not yet know which interaction model is
right. It is for product exploration: each participant must ship three
different self-contained HTML concepts and explain the tradeoffs.

This differs from `examples/design-landing/`: landing pages usually
optimize one presentation. Multi-concept UI cooks optimize divergence
and comparison across possible workflows.

Good fits:

- a dashboard workflow with unclear density;
- a learning tool where pedagogy could be wizard, sandbox, or drills;
- a map, timeline, table, or command-center surface;
- a first-pass mobile or desktop app concept.

## Adapt

Replace the raw files with the product, workflow, and data. Make
`workflow.md` specific enough that judges can tell whether a concept
works, but leave room for participants to choose different structures.
